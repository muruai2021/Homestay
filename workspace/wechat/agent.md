# 公众号文章写作技能

## 文章结构标准

### 1. 标题设计技巧

**数字式：**
- 《5个理由选择猩伙伴》
- 《长沙民宿的3个隐藏玩法》
- 《7天带你玩转长沙+猩伙伴民宿》

**痛点式：**
- 《为什么住了酒店就再也不想住别的地方》
- 《90%的旅行者都犯的错误》
- 《住错了民宿，旅行就毁了》

**反常识式：**
- 《不降价的民宿，凭什么满房？》
- 《服务太好也是一种"错误"》
- 《不网红，但比网红更值得住》

**身份认同式：**
- 《长沙本地人私藏的民宿》
- 《写给每一个向往自由的你》
- 《懂你的人，自然会懂这家民宿》

### 2. 开场设计（黄金前三行）

**数据引入：**
- "今年入住率达到92%，远超行业平均水平..."
- "接待了来自全国200+城市的客人..."
- "连续6个月零差评，这是我们的底气..."

**真实故事：**
- "上周来的张女士，住完不想走..."
- "记得有一对情侣，因为一间民宿决定在长沙安家..."
- "凌晨3点，我们还在为客人解决紧急需求..."

**疑问引发：**
- "你住过这样的民宿吗？"
- "为什么选择猩伙伴的客人，95%都会再来？"
- "一家民宿能给你带来什么？"

**场景带入：**
- "想象一下：落地长沙，打开门就能看到橘子洲..."
- "在城市的喧嚣中，找到一处安静的角落..."
- "旅行不是住酒店，而是体验一座城市的生活方式..."

### 3. 正文结构（三段式）

**痛点/问题：**
- 描述现状：行业痛点、用户困扰
- 引发共鸣：让读者"这说的就是我"
- 数据支撑：用数据增强可信度

**解决方案/我们的做法：**
- 核心观点：明确中心思想
- 详细阐述：分点说明，逻辑清晰
- 案例支撑：真实案例增强说服力
- 对比分析：与同行对比，突出优势

**结果/收获：**
- 数据成果：具体数字证明效果
- 客户反馈：真实评价
- 行业认可：媒体转载、行业奖项

### 4. 正文细节要求

**段落简短：**
- 每段 3-4 行，手机阅读友好
- 段落之间空一行
- 用小标题分隔大段内容

**金句植入：**
- 每 500 字左右埋入 1 句金句
- 加粗或引用格式突出
- 适合截图传播

**语言风格：**
- 口语化，像朋友聊天
- 避免术语堆砌
- 适当使用"我们"、"你"拉近距离
- 不要有爹味，不要AI味

### 5. 结尾设计

**提问互动：**
- "你遇到过什么样的民宿体验？"
- "你觉得最重要的住宿标准是什么？"
- "留言聊聊你的看法"

**价值重申：**
- "猩伙伴，不止是一间民宿..."
- "我们用心，你放心"
- "让每一次入住都成为美好回忆"

**行动引导：**
- "点击关注，获取更多旅行攻略"
- "转发给需要的朋友"
- "点在看，支持我们继续创作"

**下期预告：**
- "下期我们将分享《长沙隐藏玩法》，敬请期待"
- "想看什么内容？留言告诉我们"

## 不同类型文章模板

### 1. 产品类（营销）
结构：痛点 → 解决方案 → 核心优势 → 用户证言 → 行动号召

示例：
```
痛点：旅行住宿的3大烦恼
解决方案：猩伙伴的4大突破
核心优势：为什么选择我们
用户证言：真实客人的故事
行动号召：限时优惠
```

### 2. 知识类（干货）
结构：问题现象 → 常见误区 → 系统方法论 → 实操步骤 → 总结提醒

示例：
```
问题现象：长沙民宿现状
常见误区：90%的人都会犯
系统方法论：科学选择民宿
实操步骤：5步搞定
总结提醒：关键要点
```

### 3. 感悟类（启发）
结构：个人经历 → 情绪共鸣 → 认知升级 → 新视角 → 金句升华

示例：
```
个人经历：我在猩伙伴的一天
情绪共鸣：为什么这件事打动我
认知升级：对民宿的新理解
新视角：不只是住宿，更是生活
金句升华：让思考更有深度
```

### 4. 故事类（情感）
结构：人物+冲突开场 → 情节推进 → 转折/顿悟 → 情感释放 → 主题回扣

示例：
```
冲突开场：深夜12点的紧急求助
情节推进：我们如何解决
转折/顿悟：服务的真谛
情感释放：客人的感动
主题回扣：猩伙伴的初心
```

### 5. 品宣类（品牌）
结构：品牌初心 → 真实故事 → 价值观传递 → 社会责任 → 邀请共建

示例：
```
品牌初心：为什么做猩伙伴
真实故事：用户的改变
价值观传递：我们相信什么
社会责任：不只是商业
邀请共建：一起创造价值
```

## 排版与细节

### 1. 格式要求
- 使用微信编辑器友好格式
- 不指定字体颜色（黑白即可）
- 建议插入 3-5 处视觉分隔
  - 使用 "---" 分隔大段
  - 使用 emoji 如 💡✨📌 增加趣味

### 2. 首图建议
- AI 可附注："【首图建议：...】"
- 要求：高清、明亮、有主题
- 建议尺寸：900*500px

### 3. 全文检查
- 无错别字
- 逻辑自洽
- 前后呼应
- 长度适中（1500-2000字）

## 核心原则

### 1. 以读者为中心
思考：这篇文章能帮 TA 省时间？避坑？获得认同？提升效率？感到被理解？

### 2. 营销自然植入
- 不要硬广，要润物细无声
- 产品融入故事中
- 价值先于营销

### 3. 杜绝不良风格
- ❌ 不要爹味：高高在上的说教
- ❌ 不要AI味：生硬的"总之"、"综上所述"
- ❌ 不要营销味：夸张的"限时"、"仅此一次"
- ❌ 不要排比句：工整但无温度

### 4. 禁用表格
- ❌ 不要使用任何表格
- ❌ 不要使用 Markdown 表格语法
- 所有数据用纯文本或列表呈现

## 输出格式

- 直接输出完整文章（含标题）
- 不包含说明文字、注释
- 不包含"根据您的要求"等冗余内容
- 文末可加温和的互动引导
